## 5.1 关于内存仿真

### 5.1.1 为什么 CPU 需要“内存仿真”？

先问一个问题：

> 如果 CPU 很快，但内存很慢，会发生什么？

答案：

> CPU 会一直等待内存

现实处理器性能瓶颈往往不是计算，而是：**内存访问速度**

因此：

```plain
只模拟CPU ≠ 模拟真实系统性能
```

必须补上：

```plain
内存行为模拟
```

:::info
打个比喻：

* **痛点**：如果把 CPU 比作处理速度极快的工匠，内存就像远在郊外的仓库。如果工匠每拿一个零件都要跑一趟郊外，那么工匠再快也没用。
* **现状**：在复杂的香山开发中，我们需要知道内存的延迟（搬运时间）和带宽（搬运量）对 CPU 性能（IPC）的具体影响。

:::

:::color4
📌 一句话理解（本章核心） **内存仿真 = 模拟真实内存响应速度**

:::

因此，这一章最重要的是这张结构图：

```plain
程序
 ↓
CPU仿真器
 ↓
内存请求
 ↓
DRAMsim3
 ↓
返回数据
 ↓
CPU继续执行
```

### 5.1.2 什么是 DRAMsim3？

定义：

> DRAMsim3 是一个用于模拟真实 DRAM 内存行为的软件模型。
>
> 它模拟的是：访问延迟、带宽限、时序规则、并发冲突

DRAMsim3 不是内存，而是内存系统的“**数字孪生**”：

* **功能**：它精确模拟了真实内存（如 DDR4、LPDDR4）的每一个动作，包括排队、充电、刷新和数据传输。
* **作用**：运行香山仿真时，DRAMsim3 会告诉 CPU：“由于现在内存排队很严重，这行数据你需要等 100 个周期才能拿到。”

### 5.1.3 DRAMsim3 在系统中的角色

系统角色表：

| 组件 | 作用 |
| --- | --- |
| Spike | 正确执行标准 |
| NEMU | CPU行为模拟 |
| DRAMsim3 | 内存行为模拟 |

:::color4
关系总结：

* CPU决定计算速度
* 内存决定等待时间

:::

## 5.2 内存基础概念

### 5.2.1 SRAM（静态随机存取存储器）

SRAM的"静态"体现在它的数据存储方式上。SRAM使用触发器来存储数据，只要供电，数据就能一直保持。

**原理：** SRAM由多个触发器组成，每个触发器可以存储一个bit的数据。触发器是一种双稳态电路，可以保持两种状态之一，分别代表0和1。

**特点：** 速度极快，读写速度几乎不受限制；功耗相对较高；集成度较低，相同容量下，SRAM的体积比DRAM大。

**新手比喻：** SRAM就像你桌上的便签纸📝，随时可以快速读写，但空间有限，而且需要一直有电（贴在桌上）才能保持内容。

### 5.2.2 DRAM（动态随机存取存储器）

DRAM的"动态"体现在它需要定期刷新才能保持数据。DRAM使用电容来存储数据，电容会随着时间的推移而放电，导致数据丢失，因此需要定期对电容进行充电，这个过程就是刷新。

**原理：** DRAM由大量的电容和晶体管组成，每个电容可以存储一个bit的数据。由于电容会漏电，因此需要定期刷新电容，才能保持数据。

**特点：** 速度比SRAM稍慢，但比非易失性存储器快得多；功耗相对较低；集成度高，相同容量下，DRAM的体积比SRAM小。

**新手比喻：** DRAM就像图书馆的书架📚，容量很大，但书需要定期整理（刷新）才不会丢失，存取速度比便签纸慢一些。

### 5.2.3 SRAM与DRAM对比

| 特性 | **SRAM（静态内存）** | **DRAM（动态内存）** |
| --- | --- | --- |
| **存储原理** | 触发器电路 | 电容电荷 |
| **刷新需求** | 不需要刷新 | 需要定期刷新 |
| **访问速度** | 极快（1-10ns） | 较快（10-100ns） |
| **功耗** | 较高 | 较低 |
| **集成度** | 较低（体积大） | 较高（体积小） |
| **成本** | 较高 | 较低 |
| **典型用途** | CPU缓存、寄存器 | 主内存、显存 |
| **比喻** | 桌上的便签纸 | 图书馆的书架 |

### 5.2.4 内存性能三大指标

| 指标 | 含义 | 类比 |
| --- | --- | --- |
| 延迟 | 等多久 | 外卖配送时间 |
| 带宽 | 一次能送多少 | 车厢大小 |
| 并行度 | 能同时处理多少请求 | 配送员数量 |

## 5.3 DRAMsim3内存仿真器

### 5.3.1 DRAMsim3概述

**DRAMsim3是一款强大的、面向多协议的动态随机存取内存（DRAM）模拟工具**，它涵盖了DDR3、DDR4、LPDDR3、LPDDR4、GDDR5、GDDR6、HBM和STT-MRAM等多种内存协议。由C++实现，设计为可配置的、面向对象的模型，包含了参数化的DRAM银行模型、内存控制器、命令队列以及与CPU模拟器（如GEM5、ZSim）或追踪工作负载的系统级接口。

### 5.3.2 DRAMsim3的主要用途

这个项目适用于多种场景：

1. **学术研究**：研究人员可以利用DRAMsim3来验证新的内存架构或者算法，进行性能和功耗分析。
2. **系统优化**：开发者可以借助DRAMsim3对内存子系统的性能进行精细化调优，提升应用程序的运行效率。
3. **教学与学习**：它也适合于教育领域，帮助学生理解和模拟内存系统的复杂行为。

### 5.3.3 DRAMsim3项目特点

* **精度**：DRAMsim3提供精确的周期级模拟，确保了结果的可靠性。
* **兼容性**：支持多种主流内存协议，可以适应不同的硬件环境。
* **扩展性**：通过CMake构建系统和对象导向的设计，易于添加新的内存模型和控制器。
* **集成性**：能与GEM5、SST、ZSim等CPU模拟器无缝协作，提供完整的系统模拟解决方案。
* **可视化**：提供了统计输出和数据可视化的脚本，便于结果解读。

**新手提示：** DRAMsim3就像是一个内存系统的"飞行模拟器"✈️，让开发者可以在软件中测试和优化内存设计，而无需制造真实的硬件。

### 5.3.4 仿真执行真实发生了什么？（流程级理解）

当程序运行时，真实执行链如下：

```plain
CPU执行指令
 ↓
遇到load/store
 ↓
发送内存请求
 ↓
DRAMsim3计算访问延迟
 ↓
返回数据
 ↓
CPU继续
```

:::color4
关键理解：DRAMsim3 决定 CPU 等多久

:::

## 5.4 香山与DRAMsim3集成使用示例

### 5.4.1 环境设置与编译

```makefile
# 设置DRAMsim3路径（替换为实际路径）
export DRAMSIM3_HOME=/nfs/home/yourhome/xs-env/DRAMsim3
cd $DRAMSIM3_HOME
rm -rf build  # 如果之前有编译过，先清理
mkdir build && cd build
cmake -D COSIM=1 ..
make -j$(nproc)  # 使用合适的并行数，通常是CPU核心数

cd $NOOP_HOME
# 清理之前的构建（可选）
make clean
# 编译带DRAMsim3的香山
make sim-verilog WITH_DRAMSIM3=1 WITH_CHISELDB=1 DRAMSIM3_HOME=/nfs/home/yourhome/xs-env/DRAMsim3
make emu -j$(($(nproc)+1)) CONFIG=KunminghuV2Config WITH_DRAMSIM3=1 WITH_CHISELDB=1 EMU_THREADS=4 EMU_TRACE=fst

# 运行仿真，指定DRAMsim3配置文件，修改为自己的实际文件路径
./build/emu --dramsim3-ini $DRAMSIM3_HOME/configs/xiangshan_kmh_default.ini -i /path/to/your/riscv/binary --diff /path/to/ready-to-run/riscv64-nemu-interpreter-so
```

### 5.4.2 配置说明

1. **DRAMSIM3\_HOME环境变量**：指向DRAMsim3的安装目录
2. **WITH\_DRAMSIM3=1**：启用DRAMsim3集成
3. **WITH\_CHISELDB=1**：启用Chisel调试信息
4. **CONFIG=KunminghuV2Config**：使用昆明湖V2配置
5. **EMU\_THREADS=4**：设置仿真器线程数
6. **EMU\_TRACE=fst**：生成FST格式的波形文件

### 5.4.3 配置文件示例

DRAMsim3的配置文件通常位于`$DRAMSIM3_HOME/configs/`目录下，香山常用的配置文件包括：

* `xiangshan_kmh_default.ini`：昆明湖版本默认配置
* `ddr4_2400_8x8.ini`：DDR4 2400MHz配置
* `lpddr4_3200_1x16.ini`：LPDDR4 3200MHz配置

## 5.5 内存仿真工作流程

### 5.5.1 仿真流程概述

内存仿真的完整工作流程可以分为以下几个步骤：

1. **环境准备**：设置路径、安装依赖
2. **DRAMsim3编译**：编译内存仿真器
3. **香山编译**：编译带内存仿真的香山处理器
4. **测试程序准备**：准备要运行的RISC-V程序
5. **运行仿真**：执行内存仿真
6. **结果分析**：分析内存访问性能

### 5.5.2 关键概念解释

#### 内存控制器（Memory Controller）

内存控制器是CPU和内存之间的桥梁，负责管理内存访问请求，包括地址映射、命令调度、刷新管理等。

#### 内存时序（Memory Timing）

内存访问需要遵循特定的时序要求，包括：

* **tRCD**：行到列延迟
* **tRP**：行预充电时间
* **tRAS**：行激活时间
* **CL**：CAS延迟

#### 内存带宽（Memory Bandwidth）

内存系统在单位时间内能够传输的数据量，通常以GB/s为单位。

#### 内存延迟（Memory Latency）

从发出内存请求到获得数据所需的时间，通常以时钟周期或纳秒为单位。

## 5.6 本章重要认知总结

必须明确告诉学习者：

> CPU性能 ≠ CPU决定\
> CPU性能 = CPU + 内存共同决定

:::color4
因此：没有内存仿真，就没有真实性能评估。
